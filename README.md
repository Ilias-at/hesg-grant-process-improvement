## 📌 Situation

The **Higher Education Supplementary Grant (HESG)** is an Australian Government programme reimbursing students for educational expenses via an online portal — receiving **~4,015 applications per month**.

---

## ⚠️ Problem

> *More than half of all applications were failing — not because of policy violations, but because the system was designed to fail its users.*

| Metric | Value |
|---|---|
| Submission failure rate | **53%** |
| Login bounce rate | **55%** |
| Preventable rejections | **64%** |
| Monthly users affected | **4,015** |

**Root causes identified:**
- No input validation on dates or bank details
- Personal data exposed on home screen (privacy violation)
- Receipt upload capped at 2MB — standard phone photos rejected
- Sequential upload flow forced users to abandon mid-process
- Zero duplicate claim detection

## 🔧 Approach

| Method | Output |
|---|---|
| Excel Data Analysis | Screen traffic & login KPIs (May 2019) |
| BPMN Process Mapping | 9-step As-Is journey — 5 bottlenecks mapped |
| Stakeholder Research | 10+ student & 6 staff feedback synthesised |
| BCG 2×2 Matrix | 15 initiatives evaluated across $7.8M portfolio |

---

## ✅ Recommendations

**Phase 1 — Quick Wins · $320K**
- Interactive training curriculum ($70K) → reduces 30% of support emails
- Expand file types & upload size to 3MB ($250K) → +30% successful uploads

**Phase 2 — Strategic · $1.55M**
- Virtual chatbot agent ($550K) → errors drop from 12% to <5%
- OCR technology ($600K) → 80% reduction in manual input time
- Automated reimbursement tracker ($450K) → resolves 35% of support load

---

## 📈 Expected Impact

| Before | After |
|---|---|
| 53% submission failure | **< 20% failure rate** |
| 55% login bounce | **~25% bounce rate** |
| Staff overwhelmed with error correction | **−40% operational burden** |
| 12% erroneous submissions | **< 5% with validation** |
